线程状态变化

  • NEW A thread that has not yet started is in this state.
  • RUNNABLE A thread executing in the Java virtual machine is in this state.
  • BLOCKED A thread that is blocked waiting for a monitor lock is in this state.
  • WAITING A thread that is waiting indefinitely for another thread to perform a particular action is in this state.
  • TIMED_WAITING A thread that is waiting for another thread to perform an action for up to a specified waiting time is in this state.
  • TERMINATED A thread that has exited is in this state.

A thread can be in only one state at a given point in time. These states are virtual machine states which do not reflect any operating system thread states.

中断线程以及判断中断线程的方法

  • 通过上文知道，无论在什么情况下调用，都会调用interrupt0方法设置中断标志。
  • 只有在wait,join,sleep方法情况下会抛出InterruptedException。
  • private native boolean isInterrupted(boolean ClearInterrupted)：是本地方法，通过ClearInterrupted变量判断是否重置中断状态，设置为false
  • public boolean isInterrupted()：是普通方法，由当前线程对象调用，获取中断状态，并不会重置中断状态为false。因为传入ClearInterrupted变量为false
  • public static boolean interrupted() ：是Thread的静态方法，有线程Clazz对象调用，获取中断状态，并会重置中断状态为false。因为传入ClearInterrupted变量为true.

总结：所以使用interrupted方法中断线程时，一定会设置中断表示，但在使用isInterrupted方法时，除非有特殊情况，使用对象方法会一直得到一样的结果。
